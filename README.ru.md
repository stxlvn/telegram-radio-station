# MusicmaniA Radio

*[English version](README.md)*

Круглосуточное интернет-радио на основе музыкального архива Telegram-канала,
с синхронной ретрансляцией в видеочат Telegram и веб-плеером в стиле
Material Design 3.

## Первый релиз

**Вещание**

- Непрерывный поток в MP3 (128kbps) и lossless FLAC из одного источника,
  качество выбирается слушателем
- HLS-зеркало рядом с обычным потоком — даёт плееру реальную перемотку
  (назад на 10 сек, возврат к живому эфиру), чего не позволяет обычный
  потоковый поток без Content-Length
- Параллельная ретрансляция того же аудио в видеозвонок Telegram-группы по
  RTMP, с живым видеокадром (текущий трек, очередь, кнопка подписки),
  перерисовываемым на каждом тике, а не статичной заглушкой

**Автосоставление очереди**

- Треки выбираются случайно из всего аудиоархива Telegram-канала, без
  ручного ведения плейлиста
- Фильтр по длительности (отсеивает и короткие джинглы, и часовые сборники)
- Определение эмбиента/фоновой музыки для кино (хэштеги, слова в названии,
  длительность) с полным исключением из очереди, а не просто снижением
  вероятности — при этом обычные песни с саундтреков не трогает: спорные
  случаи проверяются через жанр/стиль релиза на Discogs
- Фильтр интро/джинглов по названию
- Защита от повтора: один трек не может повториться раньше чем через 48 часов
- Треки, загруженные через админ-панель, не попадают под автоматическую
  чистку устаревшей очереди

**Обогащение метаданных**

- Многоуровневый fallback для артиста/названия: теги в файле → собственные
  поля Telegram (title/performer) → посты-описания канала (хэштеги жанров,
  строки вида «Артист — Альбом») → поиск в Discogs как последний вариант
- Сопоставление трека в Discogs по длительности с полным треклистом релиза,
  а не по названию самого релиза — иначе треку могло присвоиться название
  альбома вместо реальной песни
- Обложка ищется тем же путём, с автоматическим откатом на аватар канала

**Клип на YouTube**

- Кнопка «Смотреть клип» появляется только когда для трека реально нашёлся
  подходящий видеоролик — ссылка, не встроенный плеер
- Поиск выполняется один раз на сервере при смене трека (не в браузере
  каждого слушателя) — иначе бесплатная квота YouTube Data API кончалась бы
  за несколько часов при частой смене треков
- Отсеивает аудио-заглушки: авто-сгенерированные каналы вида
  «Артист - Topic», ролики с пометками audio/lyric video и т.п.
- При исчерпании квоты официального API — автоматический fallback на
  yt-dlp (тоже только для поиска, само видео никуда не скачивается и не
  проксируется)

**Админ-панель**

- Веб-интерфейс с паролем для загрузки треков прямо в очередь, по одному
  или пачкой
- Вставка трека сразу после конкретного элемента очереди, а не только в конец
- Изменение порядка уже стоящих в очереди треков
- Пропуск текущего трека без нарушения остальной предзагруженной очереди
- Живое обновление статуса (текущий трек, что дальше) без перезагрузки
  страницы
- Индикатор прогресса загрузки для больших файлов на мобильном интернете

**Веб-плеер**

- Адаптивная вёрстка: на десктопе — три карточки в ряд (описание канала,
  плеер, очередь), на мобильном разворачивается в полноэкранный плеер по
  тапу
- Прогресс-бар синхронизирован с серверным временем, не сбивается от
  буферизации, с автопереподключением при обрыве связи
- Интеграция с системным медиа-центром ОС — управление с экрана
  блокировки/шторки уведомлений, обложка, живой прогресс-бар на любой
  платформе с поддержкой Web Media Session API
- Переключатель качества (MP3/FLAC) с предупреждением о расходе трафика
- Счётчик подписчиков канала в реальном времени

**Инфраструктура**

- Цепочка relay-серверов в нескольких регионах, чтобы поток оставался
  доступен при выборочных ограничениях маршрутизации, с автопереключением
  на резервный источник
- Сторожевой сервис, следящий за всеми компонентами (доступность потока,
  свежесть воспроизведения, состояние каждого systemd-юнита) и
  перезапускающий зависшее

## Структура проекта

```
backend/     Наполнение очереди, публикация текущего трека, админ-панель,
             сторожевой сервис, разовые скрипты индексации/сканирования Telegram
liquidsoap/  Граф источников радио, кроссфейд, вывод в Icecast/HLS
tgstream/    Ретрансляция в Telegram по RTMP (рендер видеокадра,
             кодирование ffmpeg, обновление счётчика подписчиков)
frontend/    Веб-плеер (одна страница, без сборки)
nginx/       Конфиги реверс-прокси (источник + edge-relay)
systemd/     Юниты для всех фоновых сервисов
```

## Установка

1. Скопируйте `.env.example` в `/etc/musicbestman/env` и заполните реальными
   значениями (креды Telegram API, пароль Icecast, RTMP URL, опционально
   токены Discogs и YouTube Data API — см. комментарии в файле, что
   обязательно, а что нет).
2. Установите systemd-юниты из `systemd/` и скрипт Liquidsoap из
   `liquidsoap/`.
3. Замените плейсхолдеры IP-адресов/домена в `nginx/` на свои сервера.
4. Авторизуйте Telegram-сессию для аккаунта, от имени которого работают
   скрипты (Telethon запросит это при первом запуске) — итоговый файл
   `.session` не должен попадать в систему контроля версий, как и любой
   другой секрет.
